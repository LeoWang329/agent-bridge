## 0. 核心决定与改动总览

**核心决定(用户已拍板):两种模式硬分离,不做单场混合。**
一场圆桌开场即定模式、全程不切换;模式写进 `run:started`,使每场的效果可单独归因。曾讨论过的「按分歧线程路由、单场混合」方案**明确不采纳**(现在)——理由:裁决版效果未被观察过,混合后无法区分两个版本各自的贡献;待两种模式各自积累若干场 dogfood 实证后,再议是否引入混合。

## 1. 动机:为什么分,为什么现在不融合

### 1.1 结构诊断(用户观察,经核对协议文本成立)

现有收敛轮的组织单元是**分歧**:简报=分歧索引(立场 A → `rounds/p2-r0.md`、立场 B → `rounds/p3-r0.md`),任务措辞=「先 steelman 对方最强形式,再反驳/收敛/标真取舍」,红队攻共识清单。这套机制的设计目标是**防假共识**(LLM 圆桌的头号病),对「怕选错」的裁决型问题是对的。

但它有一个结构性后果:**互补的、不冲突的观点被分歧索引直接过滤掉**,不经任何深挖就进主席终稿。「读了 p2 的观点,启发我延伸出一个谁都没提的点」「把 p1 的点和 p3 的点组合成更好的东西」——这类动作在协议里**没有通道**。轮次预算(实质轮 ≤3)全部花在裁决上,零花在增殖上。对「怕漏点」的探索型问题(产出是方案/报告/评估,不是裁决),这是内耗。

### 1.2 为什么硬分离而不是场内按线程路由

- **归因**:裁决版跑过的真实场次少(v2 复盘那一场为主),效果画像不完整;探索版是全新形态。混在一场里跑,复盘时无法回答「这场的价值来自哪个机制」。分开跑,每场 `run:started.mode` 定死,场与场可比。
- **姿态污染**:同一席同一轮混发「红队攻 X」和「温和延伸 Y」,对抗姿态易串味。分场后不存在此问题。
- **可逆**:分立是融合的前置,不是对立。两模式共享 R0 盲发与全部基础设施,将来若实证支持混合,按线程路由的方案随时可加回来(本提案 §5 留了口)。

### 1.3 两种模式的本质区别(一张表)

| | **裁决模式 adjudicate**(现状) | **探索模式 explore**(新增) |
|---|---|---|
| 产出形态 | 一个**决定**:选 A/B、方案能不能上、该不该做 | 一份**文档**:方案、报告、评估、考量清单 |
| 典型问句 | 「哪个好 / 能不能 / 该不该」 | 「有哪些 / 怎么做全面 / 帮我把 X 想透」 |
| 致命失败 | **选错**(假共识把错的抬进终稿) | **漏点**(盲区没人提,覆盖不全) |
| 分歧的角色 | 障碍——互斥立场,必须裁掉一个 | 矿藏——互补切面,都可能进方案 |
| 收敛轮形态 | 对抗轮(steelman + 反驳 + 红队) | 启发轮(延伸 + 甄别 + 守界) |
| 席位间关系 | 互为对手盘(论证层) | 互为燃料(论证层) |
| 终结判据 | 裁出结果 / 升级问人 / 轮上限 | 连续 2 轮无新论点(信息增量枯竭)/ 轮上限 |
| 反谄媚形态 | 红队 + steelman-翻转测试 | 甄别必须给理由 + 「多数不是理由」 |

## 2. 逐项设计

### 2.1【P0】模式定义与冻结原则

**规则**(SKILL.md 开头「这是一场匿名圆桌」引言块之后新增一小节):

> ## 两种模式(开场即定,全程不换)
>
> 圆桌有两种模式,**开场时由主席判定并写进 `run:started.payload.mode`,全程不切换、不混合**:
> - **`adjudicate`(裁决模式)**:现有全部流程——盲发 R0 + 对抗收敛轮(分歧索引 / steelman / 红队攻共识)+ 拓扑可选(blind-panel / author-critics)。适用于「怕选错」的问题。
> - **`explore`(探索模式)**:盲发 R0 + 启发轮(延伸 / 甄别 / 守界)+ 主席综合。适用于「怕漏点」的问题。见 §探索模式。
>
> 两模式共享:R0 盲发、匿名 + 字节直传、coverage 记账、事件流 / run-dir、viz、席位权限 / worktree、人在环升级、预算硬顶。**分立的目的的是效果归因**——每场只跑一种模式,复盘时才知道哪套机制在起作用;在两模式各自积累足够实证前,不引入单场混合。

**冻结原则**:本提案对裁决模式的所有既有文本**零改动**(§3 不改清单)。探索模式以**新增小节 + 新增角色文件**的方式落地,diff 与裁决路径不相交——保证基线干净。

**验收**:`git diff` 显示 SKILL.md 对 §反谄媚、§发言顺序、§信息传递等既有裁决机制小节无改动(仅新增小节 + §何时开圆桌扩写选模闸)。

### 2.2【P0】选模闸:主席按问题形态判定

**规则**(SKILL.md §何时开圆桌,在「值得开/别开」闸之后新增,现有「可选拓扑 1+2」小节整体归入裁决模式名下):

> ### 选模式:过了「值不值得开」的闸,再判「开哪种」
>
> 按**用户问题的产出形态**判:
> - 要一个**裁决**(哪个好 / 能不能 / 该不该;有限选项里选;选错代价高、不可逆)→ `adjudicate`。
> - 要一份**文档**(有哪些 / 怎么做全面 / 把 X 想透;产出是方案、报告、评估、考量清单;漏点比选错更致命)→ `explore`。
> - **判不准 → 用 `AskUserQuestion` 问用户**,两个选项各配一行说明(「裁决:帮你在几条路里选一条并给判据」vs「探索:帮你把这个问题的考量面铺全挖深」)。别猜——模式定错整场浪费,而问一次成本极低;且模式是效果归因的实验变量,该由人定。
> - 模式判定的理由一句话写进 `run:started.payload.modeRationale`(与调度 rationale 同理:判断透明可复盘)。
>
> **R0 后发现错配怎么办(不切换,但别硬跑)**:两模式的 R0 完全同形(盲发、答同一中立框设),所以错配总是在 R0 收口后暴露——例如探索场的 R0 显示核心其实是一个互斥岔路,或裁决场的 R0 显示各席观点根本互补、没有真对立。此时**不就地切换模式**(破坏归因),而是 `AskUserQuestion` 升级:「R0 显示题型与所选模式不符,是①终止本场、换模式重开(可复用本场 R0——它是盲发产物,天然模式无关,新场 `run:started.payload.importedR0From` 记来源),还是②按原模式继续跑完?」由用户拍板。复用 R0 时只需把 `rounds/p*-r0.md` 复制进新 run-dir,收敛轮全部重跑。
>
> 【2026-07-15 双评修订,见 §7:**R0 跨模式复用已废除**——R0 产出被各模式角色塑形(立场式 vs 延伸式),「盲发产物天然模式无关」不成立,且外来 R0 无法与新场事件流/`[COVERAGE-ERR]` 反审衔接;错配升级改为「换模式全新重开含重跑 R0」,`importedR0From` 字段随之删除。】

**验收**:下一场圆桌 `run:started.payload` 含 `mode` + `modeRationale`;若发生错配升级,transcript 里有对应 `human:asked/answered`。

### 2.3【P0】探索模式流程(SKILL.md 新增 §探索模式)

**规则**(新增独立小节,与 §发言顺序(裁决)平行):

> ## 探索模式(mode:"explore"):启发轮编排
>
> **目标**:靠众席之力产出一份**更宽、更深、更全面**的方案/报告——观点的并集加深挖,不是裁决。席位互为燃料,不互为对手盘。
>
> **分轮结构**:
> - **Round 0 — 盲发并行发散**(与裁决模式完全同形):所有席并行、互不可见、答同一份中立框设(`briefs/r0-framing.md`)。独立信号最纯的一轮,防锚定的根——探索模式同样需要它,否则第一个发言者定调、后面全在附和延伸。
> - **Round 1..k — 启发轮**:每席读**其他席的匿名全文**(`rounds/`,字节直传原件,机制与裁决模式完全相同),任务是三件事:
>   1. **延伸**:「这些观点让我想到什么**别人没提的新点**?哪两个观点(含我自己的)能**组合**出更好的东西?哪个点值得**再深挖一层**(给出深挖结果,不是喊口号)?」每个延伸点标注来源(「受 p2 的 X 启发」)——让交叉授粉可审计。
>   2. **甄别**:对读到的他人关键观点逐条表态**采纳 / 部分采纳 / 拒绝,均须给理由**;「多数席都这么说」不是理由。发现事实错误或逻辑漏洞直接点出(一两句,不展开成对抗轮)。
>   3. **守界**:每个延伸点须能一句话回连到 `r0-framing.md` 的原始问题;回连不上的放 `[OUT-OF-SCOPE]` 停车场一句话带过,不展开。**宁可停车场长,不可正文蔓延。**
>
> **主席在启发轮做什么**(纪律与裁决模式同源:只索引/记账,不复述实质):
> - **素材索引**(替代裁决模式的分歧索引):`briefs/rN-brief.md` 列观点线索的中立标签 + 指向 `rounds/` 全文的指针 + 每席的延伸任务(如「线索 C 与你 R0 的第 2 点相邻,深挖交集」)+ 一行边界重申(原始问题一句话)。**不含共识靶单、不派红队**——那是裁决模式的机制。
> - **coverage 记账**照旧(每收敛轮 append `coverage`,上轮每席 included/parked/irrelevant + 理由);席位 `[COVERAGE-ERR]` 反审照旧。
> - **深度调度**:发现某条线索连续两轮只被「提及」没被「挖深」,下一轮点名一席专挖它(写进 `schedule:decision.rationale`)。宽度自然涌现,深度要主席逼。
>
> **互斥岔路冒出来怎么办(探索场内不裁)**:启发轮中若暴露真正互斥的技术岔路(A 成立则 B 不成立),主席 append `disagreement`(`kind:"fork-parked"`),**不组织对抗轮**:
> - 若本质是产品/价值取舍 → 走既有 needs-human 升级(§人在环);
> - 若是可技术裁决的岔路 → 终稿 `final.md` 单列「未裁决岔路」一节(双方论证的指针 + 主席对影响面的一句评估),建议用户**另开一场裁决模式圆桌**(可复用相关席位的匿名论证作输入)【§7 修订:措辞收窄为「附作新场议题材料(如 1+2 被攻立场),怎么用由新场手册定」——不是 R0 复用】。
> 这是分立的代价,明码标价:探索场不假装裁决,裁决需求交给裁决场。
>
> **终结条件**(照抄现有,它天生按信息增量判):连续 2 轮无新论点(主席判「这轮有没有引入新考量」)即停;实质轮 ≤3;预算硬顶 N×(1+3) 轮。
>
> **终稿**:主席综合**全部**素材(R0 + 各启发轮的延伸与甄别 + 停车场 + 岔路)成 `final.md`:结构建议为「方案主体(按主题聚合各席贡献,标注来源席)/ 甄别结果及理由【§7 修订:原「已甄别拒绝的观点及理由」——告警=缺理由/整节缺失,零拒绝合法】/ 未裁决岔路 / 越界停车场(留给用户自己判断要不要另立议题)/ 需人拍板项」。主席注记(completeness critic)照旧可加。**席位的代码事实断言进终稿前抽查 1–2 条**(§主席自查,两模式通用)。

**验收**:一场真实探索圆桌的 `briefs/` 无共识靶单、无红队任务;`rounds/` 启发轮产出含带来源标注的延伸点与甄别表;`final.md` 的「甄别结果及理由」一节逐条带独立理由、承重结论有质疑/核验记录【§7 修订:原验收「含已拒绝观点节」与不设拒绝配额冲突,已废除——**零拒绝合法**,查的是理由与核验覆盖,不是拒绝数】。

### 2.4【P0】新角色 `roles/panelist-explore.md`

新增文件,骨架从 `panelist.md` 派生:**匿名纪律、read/write 档说明、SHARED-PRINCIPLES 块、「用文件别等人喂」、[COVERAGE-ERR] 反审、不读 transcript、无视风格猜测——全部原样保留**;差异只在核心纪律段与产出格式:

> ## 启发纪律(本角色的核心 —— 探索圆桌的价值全在这)
>
> - **用他人观点当燃料,不当靶子**:读别席匿名全文时,你的任务不是反驳,是回答三个问题——「这让我想到什么别人没提的新点?」「哪两个观点(含我自己的)能组合出更好的东西?」「哪个点值得再深挖一层?」深挖要给出结果(具体展开到能落地),不是喊「这里值得深挖」。每个延伸点标注来源(「受 p2 的 X 启发」)。
> - **甄别,不照单全收(反谄媚的探索版)**:采纳或拒绝他人观点都必须给理由;「多数席都这么说」不是理由。发现他人观点有事实错误/逻辑漏洞,直接指出(一两句点破即可,不展开成对抗)——你不是红队,但也不是复读机。**全盘采纳所有人的所有观点 = 本角色的失败**(那说明你没甄别)。
> - **守界**:每个延伸点必须能一句话回连到框设的原始问题;回连不上的放 `[OUT-OF-SCOPE]` 停车场一句话带过,不展开。宁可停车场长,不可正文蔓延。
> - **深挖优先于铺新**:一个点挖到能落地,好过五个点各喊一嗓子。新点当然欢迎,但每轮至少有一处「往深走了一层」的展开。
> - **主动补盲区**(与裁决版同):说出别人漏掉的约束、边界情况、被忽视的风险。
> - **真取舍识别**(与裁决版同):无客观优劣的取舍明说「该由人拍板」,标进「需人拍板」小节,不硬造正确答案。
>
> ## 产出格式(你的最后一条回复 = 会被主席读取的发言)
>
> - **新增与延伸**:每点 = 来源标注(受谁的哪个点启发 / 自发)+ 内容 + 为什么重要。
> - **深挖**:本轮重点深挖的 1–2 个点的具体展开(落到可执行/可验证的粒度)。
> - **甄别表**:对读到的他人关键观点逐条:采纳 / 部分采纳 / 拒绝 + 理由。
> - **盲区与风险**:别人可能漏掉的。

### 2.5【P1】探索模式的防退化(已知风险,dogfood 观察点)

探索模式砍掉了红队与 steelman——反谄媚防线变薄,这是**有意为之的实验变量**,但要睁眼看着它:

| 风险 | 场内防线 | dogfood 观察点 |
|---|---|---|
| **礼貌延伸传播错误观点**(某席的错误断言被别席当燃料延伸,错误进终稿) | ①甄别必须给理由,全盘采纳=角色失败;②主席事实抽查照旧(§主席自查);③终稿单列「已拒绝观点及理由」,拒绝率为零 = 警报 | 复盘时抽查终稿 2–3 条承重结论,回溯它在哪轮被谁甄别过;若从未被质疑,记一次「甄别缺席」 |
