## 1. 证据基础（为什么是这几刀）

本场量化数据：

| 席位 | R0 耗时/产出 | R1 耗时/产出 | contextUsage r0→r1 |
|---|---|---|---|
| p1 omp/deepseek | 134s / 5.9k 字 | 210s / 6.2k 字（红队） | 36k→50k |
| p2 codex | 373s / 3.1k 字 | **46s / 1.7k 字**（steelman-翻转） | 88k→114k |
| p3 claude | 273s / 6.1k 字 | 195s / 4.4k 字（终裁） | 317k→**464k**（虚高~2×，见 BUG 文档） |

三个关键实证：

1. **共识裸奔**：5 条「强共识」（中间件零改动、不限流、匿名可读、单表、SSO 解耦）从未被对抗。复盘抽查发现三席都「核实」了鉴权路径、**无一席提 CSRF**——评论 POST 是纯 cookie 鉴权 route handler，恰好 `src/auth/cookies.ts` 设了 `sameSite:'lax'` 才无事。**运气，不是流程的功劳。** 异引擎读同一份代码、拿同一张「查鉴权、查 XSS」检查单 → 共享同一盲区；引擎多样性 ≠ 视角多样性。
2. **空转半轮**：三席 R0 的「需人拍板」小节**都已独立**把「先审 vs 先显」标成政策取舍，R1 却又花一轮「确认它确实是取舍」。主席被 2:1 的立场分布带偏，把它标成 `by-merit`，忽视了更强的「全席一致 meta 判断」信号。
3. **记账劳动**：主席手工 append 23 条 transcript 事件（每条一次工具往返 + heredoc），其中 turn:produced/round:started 等机械事件占 2/3，纯可自动化。

## 2. 逐项改动：具体怎么做

### 2.1【P0】红队默认靶 = 无人质疑的共识

**规则**（加进 SKILL.md §反谄媚/防假共识，替换现有「红队探针」条目的靶向描述）：

> - **红队探针（靶向规则）**：红队的默认靶是**上一轮无人质疑的共识清单**，逐条找失效模式——若终稿有隐患，它藏在没被对抗过的共识里（异引擎共享同一检查单时共享同一盲区），不在被反复拉锯的取舍里。已被**全部席位**各自标为 needs-human 的分歧**不进红队靶单**（那是升级队列的事，见 §人在环）；分歧只有在立场分裂、且无人标 needs-human 时才值得红队攻多数方。红队产出的最低标准：对每条共识给出「我查了什么、为什么攻不动」或具体失效场景——泛泛表态不算。轮换扮演者。

**简报模板配套**（加进 §信息传递「建分歧索引 + 框任务」条目）：

> 收敛轮简报必须含「**共识靶单**」一节：上一轮无人反对的每条技术判断列成表，红队席逐条攻击。分歧索引与共识靶单分开列，别混。

**验收**：下一场圆桌的 `briefs/rN-brief.md` 里有共识靶单；红队席的 `rounds/` 产出里对每条共识有「查了什么」的记录。

### 2.2【P0】全席一致 needs-human → 直接升级 + 人机重叠

**规则**（加进 SKILL.md §发言顺序 + 终结判定，作为收敛轮开始前的「分歧分类」步骤）：

> **分歧分类（R0 收口后先做，决定谁进对抗轮、谁直接升级）**：
> - **全席一致 needs-human**（每席都在各自「需人拍板」小节独立标注同一项）→ 该分歧标 `kind:"needs-human"`，**跳过对抗轮直接进升级队列**。全体独立同判是强信号，对抗轮翻案概率低——再跑一轮多半在重推已知结论。若仍想给用户配「决策证据包」（如红队场景），把它作为**升级选项的附注**由主席自己写，不值一个席位轮。
> - **立场分裂 / 有席声称可技术裁决 / 有席没表态** → 才开对抗轮（steelman + 红队）测试真收敛 vs 礼貌投降。
>
> **人机重叠**：升级问人不阻塞席位——`AskUserQuestion` 只阻塞主席，后台席位照跑。正确顺序：先把本轮席位任务非阻塞派发 → 再问人 → 答案回来 `wait` 收口，把用户决定作为约束注入下一简报。「问完人再派活」的串行等待是纯浪费。

**验收**：transcript 里 `disagreement` 事件的 `kind` 字段与三席 R0 needs-human 小节一致；`human:asked` 的时间戳落在收敛轮 `turn:assigned` 之后、`turn:produced` 之前（证明重叠发生）。

### 2.3【P0】题型分流：熟路题用「1+2 作者-批评家」

**规则**（扩写 SKILL.md §何时开圆桌，标题改为「何时开圆桌，开什么形状的」）：

> ## 选结构：先判题型，再定拓扑
>
> **熟路题**（行业有成熟范式、解空间窄、关键未决点是产品取舍——评论、通知、权限、CRUD 变体这类）→ **1+2 作者-批评家**：
> - 1 席（最强引擎）直接产**全案**（数据模型/鉴权/边界/UI，落到能实施）；
> - 2 席带**不同透镜**（默认：安全怀疑者 + YAGNI 极简派）并行独立攻击该全案——攻方案，不发散自己的方案；
> - 主席合并修订 + 升级真取舍。
> 事件流/run-dir 结构不变（作者=R0 单席产出，批评家=R1 并行攻击）。比盲发 N 席便宜约一半，且对抗火力全部对准「成形的方案」，而不是互相重叠的发散。
>
> **宽解空间题**（≥2 条合理架构路径、新颖问题、跨模块契约、选错不可逆）→ **盲发圆桌**（R0 全席盲发 + 收敛轮），即原默认流程。
>
> **判不准** → 按熟路题起步（1+2）；批评家若炸出「方案根本方向存疑」（不止细节错，是路线错）再升级成盲发圆桌。升级便宜（作者稿变成一份 R0 输入），降级浪费。

**与 agent-bridge-dev 的边界**（同步更新 §与 agent-bridge-dev 的边界）：1+2 是**设计审议**拓扑（席位不写码、产出是方案），dev 的「实现→评审」是**代码交付**闭环；dev 的 2-agent 架构讨论仍是 2 席特例，不动。

**验收**：`run:started.payload` 增加可选字段 `topology: "blind-panel" | "author-critics"`（EVENTS.md 补一行；viz 兼容未知字段，无需改）。

### 2.4【P1】席位差异化透镜

**规则**（加进 SKILL.md §阵容与角色）：

> - **透镜注入（默认开）**：每席除 panelist.md 外，在 **R0 消息**里带一段「你的透镜」（收敛轮由主席逐条指派任务，不需要透镜续期）。默认三透镜：
>   - **安全怀疑者**：「优先找鉴权/注入/CSRF/信息泄露/边界校验的洞。别人没提的攻击面是你的 KPI。对『鉴权已封好』类断言默认不信，逐条验。」
>   - **YAGNI 极简派**：「优先砍字段/砍状态/砍层。每个抽象都要有当下需求付账；找出方案里为想象需求付的每一分钱。」
>   - **产品同理者**：「从真实使用场景往回推。找出技术上正确但用户不会这么用/不符合真实工作流的设计。」
>   透镜播的是**注意力种子**，不是立场结论——席位仍按自己的判断下结论。
> - **何时关透镜**：议题是「判断某断言真伪」「估计某个值」这类需要纯独立信号做交叉验证的 → 全席同角色、无透镜（透镜会污染盲发纯度）。设计/评审类议题默认开（覆盖 > 纯度）。

**验收**：三席 R0 产出的关注面明显分化（安全席提出其他席没有的攻击面项）。

### 2.5【P1】主席亲自终裁 + 席位反审索引 + 事实抽查

三条独立小改，同治「主席环节的弱点」：

**(a) 主席终裁**（改 SKILL.md §阵容与角色「主席」条目）：

> - 主席**亲自判收敛性**（「这个分歧能否靠论证收敛」）——你有全场最全上下文，终稿也是你写的；把终裁外包给席位并不增加中立性（综合权仍在你手），只多花一个重轮次。席位的职责是论证与互攻，不是替主席判收敛。主席保持中立指的是**不下场持方案立场**，不是放弃裁判职能。

**(b) 索引反审**（roles/panelist.md「用文件，别等人喂」加一条 + 简报模板固定第一项）：

> - **先审主席再干活**：读收敛轮简报的第一件事是核对——主席的分歧索引/共识靶单是否**漏掉或曲解**了你上一轮的关键论点？有就在发言开头先指出（一两句），再做本轮任务。

  成本≈0（席位反正要读简报），把 coverage 从主席自报账变成被审计的账，治「主席无声漏点」的真闭环。

**(c) 事实抽查**（SKILL.md §主席自查加一句）：

> - 席位的**代码事实断言**（「我 grep 过无 X」「该函数会校验 Y」）进 final.md 前，主席至少抽查关键的 1-2 条。未核实的断言要么验证、要么在终稿标注「席位断言，未复核」。

**验收**：下一场收敛轮至少一席的发言开头出现索引反审段；final.md 无未标注的未核实断言。

### 2.6【P1】派活前查上下文健康

**规则**（SKILL.md §失败模式「上下文腐化」行扩写）：

> | **上下文腐化** | 收敛轮**分派前**先看各席 contextUsage：**≥300k 的席位不派重读任务**（读多份全文、终裁类），重任务给最低占用席；≥400k 关旧开新交接（摘要进新会话 initial_prompt）。「反正是最后一轮」不是豁免理由。注意：跨后端读数口径目前不可比（claude 是整轮聚合、虚高约 2×，见 `BUG-claude-contextusage-overcounts-vs-omp-codex-2026-07-06.md`）——该 bug 修复前，claude 读数按对半折算估计真实占用。 |

### 2.7【P2】`tools/seat-turn.mjs`：一条命令收一席一轮

**新增 skill 资产** `skills/agent-bridge-roundtable/tools/`（两个零依赖 node 脚本，与 viz/serve.mjs 同风格）：

**`seat-turn.mjs`** —— 收口一个席位轮次的全部机械动作：

```
node tools/seat-turn.mjs <run-dir> \
  --seat p1 --round 0 \
  --text-ref "<桥返回的 textRef 绝对路径>" \
  --agent omp --model deepseek/deepseek-v4-pro \
  --summary "主席一句话摘要" \
  [--extra-names "MiniMax-M3,glm"]   # 本场在座模型名，并入黑名单
```

行为（一次调用替代主席现在的 3 次工具往返）：
1. `cp` textRef → `<run-dir>/rounds/<seat>-r<round>.md`（字节直传）；
2. 泄漏扫描：内置厂商黑名单（claude|opus|sonnet|gpt|chatgpt|deepseek|gemini|qwen|anthropic|openai|minimax|kimi|codex|glm|zhipu）+ `--extra-names`，**整词匹配、按文本读（等效 grep -aiwE）**；命中 → 打印命中行 + **exit 2**（不删不改，主席决定点状处理），干净 → 继续；
3. append `turn:produced` 事件（自动算 seq、ts、charCount；半行安全：解析已有 transcript 时缓冲不完整末行）。

**`rt-event.mjs`** —— 通用事件追加器（把本场主席临时手写的 helper 转正）：

```
node tools/rt-event.mjs <run-dir> <event> <round>   # payload 从 stdin 读 JSON
node tools/rt-event.mjs <run-dir> --batch           # stdin 每行一个 {event,round,payload}，批量 append
```

`rtId` 从 run-dir 目录名推导，不再要求主席传。**EVENTS.md schema 不变**——工具只是 writer 的省力封装，不是新契约。

**SKILL.md 配套改动**：§运行目录与事件契约、§整体流程里「cp textRef + grep + append」的三步描述改为指向 seat-turn 一条命令；保留手工路径作为降级说明。

**验收**：用 `viz/sample/` 的样例数据跑一遍 seat-turn（含一次故意塞引擎名的负例，应 exit 2）；下一场真实圆桌主席的工具调用数比本场（23+）降一半以上。

### 2.10【P1】分派纪律：一裁归一席、轻任务折叠、匿名不加码

**规则**（加进 SKILL.md §发言顺序「下一个谁说」启发式之后，作为其配套的分派纪律）：

> **分派纪律（治收敛轮的重复劳动与碎片轮次）**：
> - **一个分歧的裁决权只归一席**。多席各自独立裁同一个技术点（本场：「schema 枚举是否冗余」被两席各推了一遍完整论证）是并行浪费，不是双保险——真需要双保险就用显式「双评对比」（两席同题 + `wait mode:"all"` 比对分歧），而不是靠任务书含糊导致的意外重复。收敛轮简报的任务表里，每个分歧写明「归 pX 裁」。
> - **轻任务折叠，不单独成轮**。预计产出只有一两段、且不需要新调查的任务（如「读完对方论证后表态是否翻转」），不值一个完整席位轮（本场该轮 46 秒/1.7k 字）——折叠进该席同轮的相邻任务（如「steelman+表态」作为其裁决任务的第一小节），一次消息一次收口。判据：任务书只有一个动词、不需要打开新文件的 → 折叠。注意：**折叠的是轮次编排，不是机制本身**——steelman-翻转测试仍是反假共识的核心（见 §3 不改清单），只是别让它孤零零占一整轮。
> - **匿名机制维持现状、不加码**。出生即匿名 + 字节直传 + 整词泄漏扫描已是够用的组合（本场两轮扫描全空）；文风泄漏是弱信号×弱效应（panelist.md 已明令席位无视风格猜测），**不值得为它引入改写/摘要层**——那会重新引入内容失真，正是 v1 设计明确根除过的路径。匿名的实际增量主要来自盲发 R0，收敛轮的身份遮蔽是低成本兜底，保持轻量即可。

**验收**：下一场收敛轮简报的任务表中无两席裁同一分歧；无单任务书少于两个动作的独立席位轮。

## 3. 不改清单（本场验证有效，防「优化」误伤）

| 机制 | 本场证据 |
|---|---|
| **盲发 Round 0** | 三引擎独立同向「SSO 被过度解读」——单 agent 给不出的置信度信号；反锚定成本近零 |
| **steelman-翻转测试** | p2 翻转带真实理由（「我把『只显 approved』当成了跨内容类型硬规则」）——真收敛与礼貌投降的判别机制在工作 |
| **红队的诚实判决权**（允许「攻不动」） | p1 给出三个最强失效场景后判「B 站得住」，场景包进了用户决策简报 |
| **字节直传 + 出生即匿名** | 6 份 rounds/ 原件可审计，两轮泄漏扫描全空，成本几秒钟 |
| **耦合决定打包一次 AskUserQuestion** | 三个「谁能看到什么」决定一次问完，省两轮人机往返 |
| **终结纪律** | 一轮收敛即停、无空转；轮上限/预算硬顶未触发但在场 |
| **可视化一次性询问 + 自灭看门狗** | 按设计工作，无孤儿进程 |
