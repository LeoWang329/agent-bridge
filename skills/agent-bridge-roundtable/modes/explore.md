# 探索模式(mode:"explore")— 主席手册

> **加载时机**:在 SKILL.md 过完两道闸(值不值得开 → 开哪种模式)、选定**探索模式**后读本手册;全程不换模式、**不读 adjudicate 手册**(两模式硬分离,为效果归因)。本手册只含探索模式专属编排;R0 盲发、匿名 + 字节直传、coverage 记账、人在环、收口纪律、席位权限等**共用机制在 SKILL.md 与 EVENTS.md**,此处不重复。
>
> **本模式治什么**:怕**漏点**的问题——产出是一份更宽、更深、更全面的方案/报告/评估,观点的**并集加深挖**,不是裁决。席位互为**燃料**,不互为对手盘:用别人的观点启发自己延伸、组合、挖深,同时**甄别**(并非所有观点都对)、**守界**(问题边界不蔓延)。

**角色注入**:每席 `append_system_prompt_file=<base>/roles/panelist-explore.md`(绝对路径、存在、非空)。

## 分轮结构:盲发 R0 + 启发轮

- **Round 0 — 盲发并行发散**(与裁决模式完全同形,按 SKILL.md §整体流程 step 3 执行):所有席并行、互不可见、答同一份中立框设(`briefs/r0-framing.md`)。**探索模式同样必需**——没有盲发,第一个发言者定调、后面全在附和延伸,交叉授粉就退化成回声室。
- **Round 1..k — 启发轮**:每席读**其他席的匿名全文**(`rounds/`,字节直传原件,机制与裁决模式完全相同),任务是三件事:
  1. **延伸**:「这些观点让我想到什么**别人没提的新点**?哪两个观点(含我自己的)能**组合**出更好的东西?哪个点值得**再深挖一层**(给出深挖结果,不是喊口号)?」每个延伸点标注来源(「受 p2 的 X 启发」/「自发」)——让交叉授粉可审计。
  2. **甄别**:对读到的他人关键观点逐条表态**采纳 / 部分采纳 / 拒绝,均须给理由**;「多数席都这么说」不是理由。发现事实错误或逻辑漏洞直接点出(一两句点破即可,不展开成对抗轮——本模式没有红队)。
  3. **守界**:每个延伸点须能一句话回连到 `briefs/r0-framing.md` 的原始问题;回连不上的放 `[OUT-OF-SCOPE]` 停车场一句话带过,不展开。**宁可停车场长,不可正文蔓延。**

## 启发轮简报:素材索引

(共同纪律——只索引不复述实质、小索引可 inline、指针指向 `rounds/` 匿名全文——见 SKILL.md §信息传递;这里只定探索模式的**索引形态**。)

- **素材索引**:把上一轮各席观点聚成**观点线索**,每条一行**中立标签** + **指向相关 `rounds/` 全文的指针**(线索 C:见 `rounds/p1-r0.md`、`rounds/p3-r0.md`)+ 每席的明确延伸任务(如「线索 C 与你 R0 的第 2 点相邻,深挖交集」)。**无共识靶单、不派红队**——那是裁决模式的机制。
- **边界重申行**:每轮简报固定含一行原始问题的一句话表述(从 `r0-framing.md` 摘出)——给守界纪律一个每轮可见的锚。
- coverage 记账、席位 `[COVERAGE-ERR]` 反审照 SKILL.md §信息传递 执行,不因模式而免(素材索引同样可能漏人/曲解)。

## 调度:深度靠主席逼,宽度自然涌现

- **深度调度**:发现某条线索连续两轮只被「提及」没被「挖深」,下一轮**点名一席专挖它**(写进 `schedule:decision.rationale`)。席位天然倾向铺新点(显得有贡献),深挖要主席用任务书逼。
- **任务分派**:每席每轮的延伸任务在素材索引里写明(谁挖哪条线索);同一条线索允许多席从不同侧面延伸(探索模式无「一裁归一席」——没有裁决权可归,重叠延伸是冗余但无害,重叠甄别还是双保险)。
- **别混对抗任务**:本模式的席位任务只有延伸/甄别/守界三件;不指派「反驳 pX」「攻这份共识」类任务——需要对抗时说明题型判断错了,走下面的岔路/错配流程。

## 互斥岔路:场内不裁(fork-parked)

启发轮中若暴露**真正互斥**的技术岔路(A 成立则 B 不成立,不是可并存的互补切面),主席 append `disagreement`(`kind:"fork-parked"`,见 EVENTS.md),**不组织对抗轮**:
- 若本质是**产品/价值取舍** → 走既有 needs-human 升级(SKILL.md §人在环);
- 若是**可技术裁决**的岔路 → 终稿 `final.md` 单列「未裁决岔路」一节(双方论证的 `rounds/` 指针 + 主席对影响面的一句评估),建议用户**另开一场裁决模式圆桌**(可把岔路双方的匿名论证附作新场议题材料——例如按 1+2 拓扑当作被攻的成形立场;怎么用由新场的模式手册定)。

这是两模式分立的代价,明码标价:探索场不假装裁决,裁决需求交给裁决场。若 R0 就暴露核心是互斥岔路(整场题型判错),走 SKILL.md §何时开圆桌「第二道闸:选模式」的错配升级流程,不要硬跑。

## 终结条件(满足任一即停,别空转)

- **信息增量枯竭**:连续 **2 轮无新论点**(主席判「这轮有没有引入新考量」——探索模式这条就是主判据)。
- **轮上限**:round 0 之后**实质轮 ≤3**(默认硬顶)。
- **预算上限**:`N 席 ×(1+3)轮` 为硬顶,任一维触顶 → 强制综合 + 标缺口。

## 终稿结构(final.md)

主席综合**全部**素材(R0 + 各启发轮的延伸与甄别 + 停车场 + 岔路),建议结构:
1. **方案主体**:按主题聚合各席贡献(标注来源席 `p<K>`,供审计回溯);
2. **甄别结果及理由**:各席对关键观点的采纳/部分采纳/拒绝及理由汇总——告警信号是**缺理由**或整节缺失,不是零拒绝(不设拒绝配额,见下 防退化);
3. **未裁决岔路**(如有):fork-parked 清单 + 另开裁决场建议;
4. **越界停车场**:`[OUT-OF-SCOPE]` 汇总,留给用户自己判断要不要另立议题;
5. **需人拍板**:真取舍清单——**默认走 SKILL.md §人在环 当场问**;仅当用户已明确本场无人值守、或预算/轮次触顶强制综合时,才允许只标注留给用户。

主席注记(completeness critic)照旧可加;**席位的代码事实断言进终稿前抽查 1–2 条**(SKILL.md §主席自查,两模式通用)。

## 防退化(本模式砍了红队与 steelman,防线变薄是有意的实验变量——睁眼看着它)

| 风险 | 场内防线 | dogfood 观察点 |
|---|---|---|
| **礼貌延伸传播错误观点**(某席错误断言被别席当燃料延伸进终稿) | ①甄别逐条给独立理由,**无理由的采纳=角色失败**(panelist-explore.md;**不设拒绝配额**——全采纳但条条有扎实理由是合法结果,防「凑一条假拒绝过指标」);②某席甄别表零质疑 → 主席**当场处置**:下一轮给该席追加复核任务「上轮甄别未见质疑项——复核承重观点中是否有应质疑而未质疑的,给出你查了什么」;**已无下一轮**(轮次/预算触顶)则把未完成复核作为「甄别缺口」写进终稿与 `run:final.openIssues`,不静默略过;③主席事实抽查照旧 | 复盘抽查终稿 2–3 条承重结论,回溯它在哪轮被谁甄别/核验过;从未被质疑 → 记一次「甄别缺席」 |
| **广度堆砌、深度不足**(每轮新点罗列,没有一条挖穿) | ①「深挖优先于铺新」角色纪律;②主席深度调度(点名专挖) | 数 final.md 里「落到可执行粒度」的条目占比 |
| **边界蔓延** | ①守界纪律 + 停车场;②简报每轮边界重申行 | 停车场条数 vs 正文越界条数(后者应为 0) |
| **锚定回潮**(启发轮互读全文后趋同) | R0 盲发已把独立信号锁定在先;延伸点强制标来源,趋同可见 | 终稿观点集应是 R0 并集的**超集**(有新增),而非**子集**(被磨平) |

**明确不加**(保模式纯度,待实证再议):终稿前的红队式核查轮。若 dogfood 显示「礼貌延伸传播错误」频发,再考虑加一道**收尾单席核查**(一席、只查承重结论、不进轮次循环)——见提案 `docs/PROPOSAL-roundtable-explore-mode-2026-07-15.md` §5 开放问题。

## 效果归因(每场复盘记,数据全部从 transcript + rounds/ 可得)

每轮新增论点数(终结判据的副产品)、延伸点带来源标注的占比(交叉授粉是否真发生)、甄别表逐条带独立理由的占比 + **承重结论被质疑/核验的覆盖率**(不用简单拒绝率——那会诱导凑假拒绝)、终稿相对 R0 并集的净增量(超集验证)、停车场条数。积累 2–3 场后对照裁决场回答:探索模式产出的覆盖面是否显著优于「单 agent 直接写」。
