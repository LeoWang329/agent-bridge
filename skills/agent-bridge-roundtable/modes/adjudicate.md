# 裁决模式(mode:"adjudicate")— 主席手册

> **加载时机**:在 SKILL.md 过完两道闸(值不值得开 → 开哪种模式)、选定**裁决模式**后读本手册;全程不换模式、**不读 explore 手册**(两模式硬分离,为效果归因)。本手册只含裁决模式专属编排;R0 盲发、匿名 + 字节直传、coverage 记账、人在环、收口纪律、席位权限等**共用机制在 SKILL.md 与 EVENTS.md**,此处不重复。
>
> **本模式治什么**:怕**选错**的问题——解空间里有互斥路径,要裁掉错的、留下对的,假共识(礼貌附和把错的抬进终稿)是头号敌人。

**角色注入**:每席 `append_system_prompt_file=<base>/roles/panelist-adjudicate.md`(绝对路径、存在、非空)。

## 可选拓扑:熟路题的「1+2 作者-批评家」(非默认,先过 SKILL.md 第一道闸)

**盲发圆桌是默认拓扑。** 当议题确属**熟路题**(行业有成熟范式、解空间窄、关键未决点是产品取舍——评论、通知、权限、CRUD 变体这类),可选更省的 **1+2 作者-批评家**:
- **1 席作者**直接产**全案**(数据模型/鉴权/边界/UI,落到能实施);
- **2 席批评家**带不同审视角度并行**独立攻该全案**(攻方案,不发散自己的方案);
- 主席合并修订 + 升级真取舍。

事件流/run-dir 结构不变(作者=R0 单席产出,批评家=R1 并行攻击);比盲发 N 席便宜约一半,火力全部对准「成形的方案」而非互相重叠的发散。`run:started.payload` 可带可选 `topology:"blind-panel"|"author-critics"`(默认 `blind-panel`;见 `EVENTS.md`)。

**四条硬约束(违反任一 → 退回盲发圆桌)**:
1. **不是默认**。默认仍是盲发圆桌;1+2 只在你**确信是熟路题**时选。
2. **判不准 → 用盲发圆桌**(不是「按熟路起步」)。盲发 R0 的独立信号是本 skill 的核心价值,不确定时不能拿它冒险——宁可多花一半成本保住独立性。**升级便宜、降级浪费**:批评家若炸出「方案根本方向存疑」(路线错、非细节错),把作者全案当 `rounds/p1-r0.md`、另开 2 席补盲发 R0(即回到 SKILL.md §整体流程 step 3 的盲发圆桌),无需推倒重来。
3. **选作者席别纠结「谁最强」**。作者引擎强弱不决定终质(批评家会兜底);**默认选 `contextUsage` 最低的席**(与 SKILL.md §失败模式「上下文腐化」联动,把最重的产全案任务放在最空的席),或领域最匹配的席。
4. **升级逻辑替换**:1+2 的 R0 只有单一作者,**没有「全席一致 needs-human」可判**(见下 §发言顺序 分歧分类)。改为:**任一批评家**独立把某点标 needs-human → 直接进升级队列(批评家视角本就偏怀疑,单席即足以触发)。

**与 `agent-bridge-dev` 的边界**:1+2 是**设计审议**拓扑(席位不写码、产出是方案),dev 的「实现→评审」是**代码交付**闭环;dev 的 2-agent 架构讨论仍是 2 席特例,不动。1+2 是 **3 席且带对抗批评**,与「2 席够就用 dev」不冲突——**够 2 席、只要有界设计讨论 → 用 dev;要 3 席对抗攻一个成形方案 → 用 1+2**。

## 发言顺序 + 终结判定(本模式的智力核心)

**分轮结构**:
- **Round 0 — 盲发并行发散**:所有席**并行、互不可见**、答同一份**中立框设**。独立信号最纯的一轮(无锚定 = 反假共识的根)。
- **Round 1..k — 定向收敛**:主席抽出活跃分歧,**定向追问**:每个分歧让相关席先 steelman 对方最强论点再反驳。

**分歧分类(R0 收口后、开收敛轮前先做,决定谁进对抗轮、谁直接升级)**:
- **全席一致 needs-human**(每席都在各自「需人拍板」小节独立标注同一项)→ 该分歧标 `kind:"needs-human"`,**跳过对抗轮直接进升级队列**(SKILL.md §人在环)。全体独立同判是强信号,对抗轮翻案概率低,再跑一轮多半在重推已知结论。若仍想给用户配「决策证据包」(如红队失效场景),由主席自己写进升级选项的**附注**,不值一个席位轮。
- **立场分裂 / 有席声称可技术裁决 / 有席未表态** → 才开对抗轮(steelman + 红队)测真收敛 vs 礼貌投降。
- (**1+2 作者-批评家拓扑**下 R0 只有单一作者,不存在「全席一致」——改为**任一批评家**独立把某点标 needs-human 即进升级队列,见上 §可选拓扑。)

**「下一个谁说」启发式**(写进 `schedule:decision.rationale`,让判断透明可复盘):
- 优先叫(a)在**高影响未决分歧**上有立场的席 /(b)对该开放问题**领域更契合**的席 /(c)**尚未被挑战过**的席。
- **轮换起始席**,别每轮同一人先说(压首因/锚定偏置)。
- **并行**:探针打**不同**分歧、互相独立时并行(压时延且保独立);**串行**:B 必须针对 A 的**具体论断**反驳时才串行,且保持定向,不搞「读完全部再泛泛表态」(那会重新引入锚定)。

**分派纪律(治收敛轮的重复劳动与碎片轮)**:
- **一个分歧的裁决权只归一席**。多席各自独立裁同一技术点是并行浪费,不是双保险;收敛轮简报的任务表里每个分歧写明「归 pX 裁」。**逃生口**:真要双保险,用**显式**「双评对比」(两席同题 + `wait mode:"all"` 比对分歧)——别把这条误读成禁止一切双评,禁的是**任务书含糊导致的意外重复**。
- **轻任务折叠,不单独成轮**。预计只产一两段、不需新调查的任务(如「读完对方论证后表态是否翻转」)折叠进该席同轮相邻任务,一次消息一次收口。判据:任务书只有一个动词、不需打开新文件 → 折叠。**但折叠的是轮次编排、不是机制**:steelman-翻转测试仍是反假共识核心(见下 §反谄媚),折叠后的 steelman 段须有**独立小节标题、质量不低于原独轮**(主席收口时检查),否则退回独立轮。
- **匿名不加码**(防误伤、非新规):文风泄漏是弱信号 × 弱效应(SKILL.md §信息传递「残余诚实交代」),**不为它引入改写/摘要层**——那会重新引入内容失真,正是本设计要根除的。

**终结条件(满足任一即停,别空转)**:
- **收敛到纯取舍**:剩余分歧全是「无客观优劣的真取舍」→ **升级问人**(SKILL.md §人在环),别再循环。
- **重复**:连续 **2 轮无新论点**(主席判「这轮有没有引入新考量」)。
- **轮上限**:round 0 之后**实质轮 ≤3**(默认硬顶)。
- **预算上限**:`N 席 ×(1+3)轮` 为硬顶,任一维触顶 → 强制综合 + 标缺口。

## 收敛轮简报:分歧索引 + 共识靶单

(共同纪律——只索引不复述实质、小索引可 inline、指针指向 `rounds/` 匿名全文——见 SKILL.md §信息传递;这里只定裁决模式的**索引形态**。)

- **分歧索引**:每个本轮该咬的分歧一行**中立标签** + **指向相关 `rounds/` 全文的指针**(立场 A → `rounds/p2-r0.md`、立场 B → `rounds/p3-r0.md`)+ 明确任务(先 steelman 对方最强形式,再反驳/收敛/标真取舍)+ 每个分歧写明「归 pX 裁」(见上 分派纪律)。
- **共识靶单**:上一轮无人反对的每条技术判断列成表,供红队席逐条攻(见下 §反谄媚)。**共识靶单与分歧索引分开列、别混。**
- **调度变体 debate**:某分歧卡在需逐条对轰的技术点时,开一轮双方互读对方匿名全文连续对质——这只是**普通收敛轮的调度模式**(写进 `schedule:decision.rationale`),不涉及任何改写或降信息。

## 反谄媚 / 防假共识(LLM 圆桌最大暗礁,专门设计)

- **Round 0 盲发并行**(无锚定)。
- **红队探针(靶向规则)**:红队的默认靶是**上一轮无人质疑的共识清单**,不是被反复拉锯的取舍——若终稿有隐患,它藏在没被对抗过的共识里(异引擎共享同一检查单时共享同一盲区;实证:一场三席都「查了鉴权」却无一席提 CSRF,纯靠运气没出事)。收敛轮里主席**指派至少一席**逐条攻这份共识清单(读被攻击立场的完整匿名论证,弹药更足),并**轮换扮演者**。**红队产出最低标准**:对每条共识给出「我查了什么、为什么攻不动」或具体失效场景——泛泛表态不算。**只列会进终稿、且有技术风险的共识**,不是每句无争议的话。**边界**:已被**全部席位**各自标为 needs-human 的分歧**不进红队靶单**(走升级队列,见上 分歧分类 与 SKILL.md §人在环);分歧只有在立场分裂、且无人标 needs-human 时才值得红队攻多数方。这是对抗式 verify 的圆桌版。
- **引擎/模型多样性**(谄媚倾向不在异引擎间同构共享)。
- **匿名全文 + 中立分歧索引**(见 SKILL.md §信息传递):去权威(匿名)压附和,不靠"去信息"(有损摘要)——各席看得到彼此完整论证,只看不到是谁。
- **辨真收敛 vs 礼貌投降**:某席翻转立场时,追问「为何对方论点是决定性的」——给得出真实理由才算收敛,只会附和 = 假共识,不计入收敛。
