# 评审团结论(双架构师,主控综合)

## 挖出的隐性需求(已转为访谈问题/合同条款)

1. **stdout/stderr 分离语义**——CI 消费的关键;两位架构师一致:必须显式定死(→ 访谈 Q1 → 假设账本 A1)。
2. **错误与中断路径**——一位架构师指出「快乐路径的 JSON 谁都会写,合同的价值在异常路径」(→ AC3)。
3. **schema 稳定性承诺**——文档必须给出输出 schema 表,否则 CI 消费方无法演进(→ G2 AC2)。

## 分歧与裁决

- 是否引入 `--output=json|text` 通用参数替代布尔 `--json`:一方主张通用化,一方按 YAGNI 反对。
  **裁决:YAGNI,先 `--json`**;将来真有第三种格式再迁移,迁移成本被评估为低。

## 风险提示

- spinner 库默认写 stdout——大概率是第一轮验收的雷(事后:确实是,见 g1-i1 verdict)。
